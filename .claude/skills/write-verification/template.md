# VERIFICATION.md テンプレート

`.artifacts/features/<feature>/VERIFICATION.md` を以下の構成で作成する。

```md
---
feature: <feature_name>
last_updated: YYYY-MM-DD
---

# 受け入れ検証: <feature_name>

このファイルは検証ラウンドを時系列で蓄積する。再検証時は既存結果を書き換えず追記する。

## Round R: <検証範囲>

### 総合判定

- **判定**: Pass / Fail
- **検証日**: YYYY-MM-DD

### 受け入れ基準の充足状況

| 受け入れ基準 | 判定 | 根拠 |
| --- | --- | --- |
| [基準 1] | 達成 / 未達 | [テスト出力・確認手順] |
| [基準 2] | 達成 / 未達 | [テスト出力・確認手順] |

### テスト結果

- **コマンド**: `[テストコマンド]`
- **結果**: [pass / fail の件数、カバレッジ]

### 未達 / ギャップ

- [満たされていない要件] — 原因: [推定原因]、対象箇所: `path/to/file`

### 推奨アクション

- [ギャップ修正に向けた次の一手。ギャップ修正タスクの起点とする]
```
