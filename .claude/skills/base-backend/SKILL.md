# バックエンドコンテキスト

このスキルは、ワークスペースのバックエンド固有のコンテキストを一元管理する。

## 参照ファイル

| 参照      | パス           |
| --------- | -------------- |
| `backend` | `./backend.md` |

## 読み込みタイミング

以下のいずれかに該当する場合に `backend` を読み込む。

- API 設計 — REST / GraphQL エンドポイントの設計・実装
- レイヤー実装 — リポジトリ・サービス・コントローラー層の追加・変更
- DB 最適化 — N+1 解消、インデックス設計、コネクションプール調整
- キャッシュ — Redis / インメモリ / HTTP キャッシュの導入
- 認証・認可 — JWT 検証、RBAC、ミドルウェア追加
- バックグラウンド処理 — ジョブキュー、非同期処理の設計
- エラーハンドリング — API エラー構造、バリデーション、リトライ設計
- ロギング・監視 — 構造化ログ、リクエストトレーシングの追加

## 組み合わせルール

- テスト駆動実装には `flow-tdd` と組み合わせる。
- コードレビュータスクには `flow-review` と組み合わせる。
- Django プロジェクトでは `base-django` を優先し、このスキルは補助として使う。
