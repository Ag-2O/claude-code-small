# Django フレームワークコンテキスト

このスキルは、ワークスペースの Django/DRF 固有のコンテキストを一元管理する。

## 参照ファイル

| 参照           | パス                |
| -------------- | ------------------- |
| `architecture` | `./architecture.md` |
| `testing`      | `./testing.md`      |

## 読み込みタイミング

`architecture` を必ず先に読む。タスクに応じて `testing` を追加する。

| タスク                     | `architecture` | `testing` |
| -------------------------- | -------------- | --------- |
| 新機能・バグ修正の実装     | 必須           | 必須      |
| セキュリティレビュー・強化 | 必須           | —         |
| テストの追加・更新         | —              | 必須      |
| マージ前・リリース前の検証 | —              | 必須      |

**補足:**

- 認証・パーミッション・Cookie・セキュリティヘッダー・デプロイ設定が関わる場合は、`architecture` のセキュリティセクションに集中する。
- マイグレーション・依存関係・設定・公開 API に大きな変更があった後は、`testing` の検証チェックリストを実行する。

## 組み合わせルール

- `python` と組み合わせて Python 言語ルールを適用する。
- テスト駆動実装には `tdd` と組み合わせる。
- コードレビュータスクには `review` と組み合わせる。
