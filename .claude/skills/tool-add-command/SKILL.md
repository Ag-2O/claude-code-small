# コマンド追加ワークフロー

このスキルは、プロジェクトに新しいコマンドを追加するプロセスを定義する。

## フェーズ 1: 要件の把握

コマンド名と目的が渡されていない場合は確認する。
また、今までの会話の中で、コマンドの必要性や目的が明確になっているかを確認する。

- **コマンド名**: 小文字のケバブケース（例: `add-command`, `run-tests`）
- **コマンドの目的**: ユーザーがどんな操作をするために呼び出すか？
- **引数 ($ARGUMENTS)**: コマンドに渡す引数の有無と意味（フィーチャー名・ファイルパスなど）
- **委譲先**: 呼び出すスキル名またはエージェント名（例: `tool-add-command` スキル、`coder` エージェント）

## フェーズ 2: 既存コマンド参照

`.claude/commands/` の一覧を確認し、同種のコマンドを読み込んで構造を把握する。

- スキルを呼び出すコマンド: `add-skill.md` を参照
- エージェントを呼び出すコマンド: `review.md` / `tdd.md` を参照
- 引数を使うコマンド: `plan.md` / `spec.md` を参照

同名・類似コマンドが既に存在する場合は、新規作成ではなく既存コマンドの更新を提案する。

## フェーズ 3: コマンドファイルの作成

`./template.md` を読み込み、`.claude/commands/[name].md` を作成する。

### フロントマター必須フィールド

```yaml
---
description: >
  コマンドの目的を 1〜2 文で説明する。
  引数を使う場合は $ARGUMENTS をそのまま埋め込んでよい。
---
```

### ボディ構造

- **`# 指示`** セクションのみを持つシンプルな構造にする
- スキルを呼び出す場合: 「`[skill-name]` スキルを読み込み実行する」
- エージェントを呼び出す場合: 「`[agent-name]` サブエージェントを呼び出す」
- 引数がある場合: `$ARGUMENTS` をそのまま埋め込む

## ベストプラクティス

- コマンドはユーザーが直接呼び出す薄いラッパー — ロジックはスキルまたはエージェントに委譲する
- description の先頭行はコマンド一覧に表示されるため、目的が一目で分かるよう書く
- 引数が不要な場合は `$ARGUMENTS` を省略する
- 同一操作をするコマンドが既存にある場合は、新規作成せず既存を更新する
