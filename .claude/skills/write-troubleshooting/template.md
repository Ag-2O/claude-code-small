# troubleshooting/[topic].md テンプレート

`.artifacts/troubleshooting/[topic].md` を以下の構成で作成する。

```md
---
topic: <topic>
last_updated: YYYY-MM-DD
---

# トラブルシューティング: <topic>

## 根本原因 / 修正方針

[特定した根本原因と、推奨する修正方針を先に書く。代替案があれば併記する]

## 症状

[観測された問題。エラーメッセージ・スタックトレース・誤った出力・ハング・リソース枯渇など]

## 再現手順

[問題を再現する最小条件。コマンド・入力・前提状態]

## 切り分けと証拠

- [調べたこと 1: 結果・根拠（ファイル・行・ログ）とともに]
- [調べたこと 2: 結果・根拠とともに]

## 再発防止

[同種の問題を防ぐ観点。テスト設計・ガード・契約の明確化など]

## 不明点 / 次の候補

- [特定しきれなかった点]
- [次に調べるべきこと]

## 参考

- [関連ファイル・コミット・ドキュメント・URL]
```
